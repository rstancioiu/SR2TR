SR2TR - Algorithm of finding an injective mapping function between two interval-based databases

Data in Astrophysics are very often structured with the relational data model. One particularity is that every value is a real number and comes with an associated error measure, leading to a numerical interlal [value-error, value+error]. Such astrophysics databases can be seen as interval-based numerical databases. On this webpage, you can conduct experiments in order to discover an injective mapping between attributes of a source interval-based relation and a target interval-based relation. At first you choose a satisfaction (classical or canonical), then you upload two files, representing the two relations as described below. The algorithm will print the condensened representation of the two relations, the matrix of error measures and in the end the mapping found.

Exemple of file to upload (for each row, the elements are separated by a space):

A err_A B err_B C err_C
2 0.5 3 1.5 4 1.2
3 1.5 4 0.5 5 1.5

The configuration presented above corresponds to the following relation:

r A B C
t1 [1.5,2.5] [1.5,4.5] [2.8,5.2]
t2 [1.5,4.5] [3.5,4.5] [3.5,6.5]
